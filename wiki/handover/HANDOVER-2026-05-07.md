## 🧠 今天的 5 個關鍵發現

### 1. Phase 1 + Phase 2 架構分工驗證
Phase 1 routing 刻意 recall-first，會產 FPs（如 Pride Collection → iphone-17-pro）；Phase 2 LLM `target_valid=false` 正確拒絕。**兩階段設計在真實資料上運作正確**。

### 2. IDF 權重讓 routing 實用
`apple-intelligence` 在 3 個產品頁都有 → idf=0 → 不成噪音。沒有 IDF 的 routing 會讓所有文章 fan-out 到所有產品。

### 3. yaml_mini round-trip bug（Phase 5 live test catch）
Unit tests 只測 first-time write → 漏測「讀既有 frontmatter → 再寫」的循環。**live CLI run 第一次就炸**。教訓：有 serialization 的東西一定要測 round-trip。

### 4. LLM 尊重 ownership 需要 prompt + filter 雙層防護
LLM 常把 human-owned section 用 `update/append` 而非 `suggest`。靠 prompt 規則不夠，Phase 3 filter 強制降級 → review queue。

### 5. 「Files, not UI」哲學在 review workflow 很 work
Reviewer 用 `$EDITOR` 勾 checkbox，`--apply-decided` 全自動，`_archive/` 保留 audit trail。沒寫任何 UI 就完整支援人類審核流程。
